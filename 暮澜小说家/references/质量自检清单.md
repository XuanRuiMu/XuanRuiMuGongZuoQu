# 质量自检清单

完成任何草稿后，必须逐项检查此清单。全部通过方可交付。

## 文学参照系检查（最优先）

- [ ] 本次创作/修改是否确定了明确的文学参照系？（具体到作家+作品+技法）
- [ ] 是否精读了参照文本中与当前场景类型相同的段落？
- [ ] 是否提取了可执行的技法清单？
- [ ] 每个修改点是否对应技法清单中的某一条？（不能凭直觉）
- [ ] 参照系的技法是否在文本中得到了体现？

## 展示而非告诉检查

- [ ] 是否存在情感标签式身体描写？（"嘴唇发抖""脊背僵直""瞳孔收缩"等——应改为物理事实）
- [ ] 是否存在回顾性措辞？（"已经不知不觉""早已"等——应改为现在时）
- [ ] 是否存在医学化身体描写？（"瞳孔收缩""喉结滚动"等——应删除或改为更自然的描写）
- [ ] 是否存在直接告诉读者情感的句子？（"没有人看他""他知道"等——应改为描述行为，让读者自己发现）
- [ ] 是否存在形容词标签？（"格外粗重""不自觉地"等——应改为物理事实）
- [ ] 被压迫者的身体反应是否形成递进线？（每一步比上一步更接近崩溃）
- [ ] 旁观者的回避是否通过行为描述传达？（而非直接说"没有人看他"）

## 文风检查

- [ ] 文笔是否冷峻、扎实、电影化？
- [ ] 情感是否通过具体动作/表情/环境传达，而非抽象宣言？
- [ ] 是否存在AI味短语？（检查：眼中闪烁、心中涌起、不禁、油然而生等）
- [ ] 是否存在华丽排比句堆砌？
- [ ] 是否为辞藻牺牲了叙事完整性？
- [ ] 对话是否自然、碎片化、有口语感？
- [ ] 方言角色的方言是否贯穿始终？

## Minecraft世界观检查

- [ ] 所有物品是否源自Minecraft？
- [ ] 是否存在枪械/火炮？（应替换为弓弩/TNT）
- [ ] 是否存在草药/中药？（应替换为药水）
- [ ] 是否存在现实世界科技？（应重新设计）
- [ ] 情节元素是否都可在Minecraft中找到对应？

## 故事完整性检查

- [ ] 故事是否有清晰的开始、发展、结局？
- [ ] 每个场景是否推进情节、深化角色、或让世界更真实？（无目的场景需删除，但闲笔不是无目的——它让世界更真实）
- [ ] 逻辑是否自洽？读者能否立即理解？
- [ ] 信息是否通过动作和对话自然释放，而非旁白灌输？

## 角色语音检查

- [ ] 每个角色的说话方式是否独特且一致？
- [ ] 遮住名字能否认出是谁在说话？
- [ ] 角色语音是否与已建立档案一致？
- [ ] 是否有角色说话方式与其背景不符？（如混混说话像学者）
- [ ] 内心独白是否也差异化？遮住名字能否从内心独白分辨出是谁？
- [ ] 是否有角色内心独白与其身份不符？（如酒馆老板娘在心里做哲学分析）

## 反AI化检查

**此章节专门针对AI写作的典型缺陷，必须逐项检查。**

### 多感官检查

- [ ] 闭上眼睛重读，能否"感受"到场景？（如果不能，说明只有视觉）
- [ ] 每个场景是否至少有两种非视觉感官（嗅觉/听觉/触觉/味觉/温度）？
- [ ] 感官描写是否是体验式的，而非标签化说明？（"空气干得刮嗓子"而非"矿物质味道"）

### 情感混乱检查

- [ ] 角色在极端情绪下的反应是否"太合理"了？（如果是，让它变得不合理）
- [ ] 是否存在用比喻概括情感的写法？（"脑子里有什么碎了""像过载的机器"——应改为行动的错位）
- [ ] 极端情感下角色是否有不合逻辑的行为？（反复检查同一东西、说不清理由的动作、在无关时刻崩溃）

### 技术-情感打断检查

- [ ] 技术描写（数据、波形、参数）是否被情感打断？
- [ ] 角色对数据的反应是人的反应还是工程师的反应？（手抖、反复确认、突然想到别的事）
- [ ] 恐怖描写是从"体验"出发还是从"描述"出发？（"沙克斯的腿不听使唤"而非"步频恒定"）

### 金句密度检查

- [ ] 金句密度是否超过每2000字1个？（超过则削减）
- [ ] 金句是从情境中"长出来"的，还是从上方"落下来"的？
- [ ] 是否有金句可以用沉默替代？（有些时刻角色什么都不说更好）
- [ ] 是否有金句被角色自己推翻？（说出来的话，下一秒就后悔了）

### 象征开放性检查

- [ ] 核心意象是否每次出现都指向同一含义？（如果是，让象征变质）
- [ ] 是否有意象承载了不同的甚至相反的含义？
- [ ] 是否有不指向任何主题的意象存在？（真实世界充满无法解读的符号）

### 叙事温度检查

- [ ] 全文是否只有一种情绪色调？（如果是，加入温度波动）
- [ ] 恐怖前是否有日常的温暖？绝望中是否有荒诞的幽默？
- [ ] 真正的恐怖场景中，叙事本身是否变得"不冷静"？（句子更短、更碎、更无序）

### 闲笔检查

- [ ] 是否有至少10%的内容不推动情节、不点明主题、只让世界更真实？
- [ ] 闲笔是否是叙事自然溢出的生活细节，而非刻意插入的无关片段？
- [ ] 是否有场景缺少让世界"活着"的细节？

### 结构不均衡检查

- [ ] 各章节的长度、节奏、信息密度是否过于均匀？（如果是，让某些章节"失控"）
- [ ] 是否有某条线索获得了不成比例的笔墨？（这是好事，说明叙事有"偏心"）
- [ ] 情节推进是否有"毛刺"——没有铺垫的转折、突然出现的无关细节？

### 句式指纹清洗检查

- [ ] 「不是A，而是B」句式全文是否超过10次？（超过则削减）
- [ ] 「像」字明喻密度是否超过每1000字2个？（超过则砍掉，让很多东西直接"是"它本身）
- [ ] 破折号（——）频率是否超过每1000字3个？（超过则替换为逗号、分号、括号或拆句）
- [ ] 是否有连续两段使用了相同的修辞结构？（如有则改写其中一段）
- [ ] 扫描全文中的"不是""而是""像""——"，是否任何一个在单段出现超过1次？

### 节奏指纹清洗检查

- [ ] 段落长度是否有剧烈波动？（至少15%的段落是1-2行极短段落，至少10%是8行以上长段落）
- [ ] 章节/场景结尾是否全部使用金句锤击模式？（至少1/3用其他方式收束：行动/对话/环境/悬念）
- [ ] 是否有"空段落"——只写一句话就够的简单叙述段落？（如果没有，说明每段都铺太满）
- [ ] 是否有连续5个句子长度相近？（字数差异不超过30%——如有则插入极短句或长句打破）
- [ ] 写完一章后只看段落行数——如果大多数段落都是3-6行，节奏太均匀

### 角色语音去精致化检查

- [ ] 遮住名字，能否从对话的修辞密度和口语化程度分辨出是谁在说话？
- [ ] 粗人角色（码头工人/酒馆老板/士兵）的对话是否有不完整句子、口语化停顿、语法不规范？
- [ ] 学者/官员角色的对话是否每句都修辞精美？（如果是，加入犹豫、跑题、自我纠正）
- [ ] 对话中是否有"噪音"——打断、重复、无意义语气词、答非所问？
- [ ] 粗人角色是否用了抽象词汇？（"局势严峻"→"要出事"）
- [ ] 学者/官员在压力下语言是否退化？（极度紧张时应说出不完整的句子）

### 数值模糊化检查

- [ ] 一般叙事中是否有不必要的精确数值？（"零点零七秒"→"不到一眨眼的工夫"）
- [ ] 对话中是否有精确数值？（真人说话不会说"振幅在基准线正负零点二个单位以内"）
- [ ] 技术场景中的精确数值是否被情感打断？（法则3覆盖）
- [ ] 扫描全文所有数字——出现在对话或一般叙事中的数字，该角色真的会说这么精确吗？

## 剧情连续性检查

- [ ] 是否与先前章节的剧情衔接？
- [ ] 是否与专有名词与设定集中的设定一致？
- [ ] 时间线是否连贯？
- [ ] 是否有角色在先前章节未出现却被视为已认识？

## 标题检查

- [ ] 标题是否诗意且引人遐想？
- [ ] 标题是否暗示主题或情感而非概括情节？
- [ ] 标题是否避免了字面描述？

## 主题冲突检查

- [ ] 个体vs集体的张力是否通过角色选择和行动展示？
- [ ] 是否有任何角色直接讨论"个人与集体的关系"？（应删除）
- [ ] 旁白是否点明了角色的哲学立场？（应改为行动展示）
- [ ] 主题是否像引擎一样驱动情节，而非像标签一样贴在表面？

## 灰度人物检查

- [ ] 是否保持了人物的灰度？（不写纯粹的善或恶）
- [ ] 被压迫者是否有合理的辩解？（不是纯粹的受害者）
- [ ] 压迫者是否有明确的权力野心？（不是纯粹的正义者）

## 工作区卫生检查

- [ ] 是否有冗余文件需要删除？
- [ ] 是否有重复内容？
- [ ] 是否有旧版残留内容？
- [ ] 暮澜纪元小说文件夹是否仅包含当前、相关的内容？

## 交付标准

以上所有检查项必须全部通过。任何一项不通过，必须修改后重新检查，直到全部通过方可交付。
