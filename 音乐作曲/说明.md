## ✨ 功能特性

这不是简单的"写段旋律"工具，而是一套**完整的音乐创作工作流**：

| 能力维度            | 详细说明                                        |
| ------------------- | ----------------------------------------------- |
| 🎼 **完整作品创作** | 多乐章交响乐、电影配乐、游戏BGM、流行编曲       |
| 🎻 **专业配器**     | 标准管弦乐队编制(4332/4331)、音色搭配、平衡原则 |
| 🎹 **风格模仿**     | 汉斯·季默/贝多芬/威廉姆斯/德彪西等大师风格模板  |
| 📄 **标准输出**     | MIDI文件(.mid) + LABS配置指南 + 配器文档        |
| 🎓 **理论支持**     | 和声学、曲式、对位法、旋律写作等完整理论库      |

### 与其他方案对比

| 维度     | 普通AI回答             | 本 Skill                         |
| -------- | ---------------------- | -------------------------------- |
| 输出质量 | 文字描述"建议用小提琴" | 可直接导入DAW的MIDI文件          |
| 专业性   | 通用知识，可能出错     | 基于配器法和作曲理论的规则系统   |
| 风格控制 | "可以试试浪漫风格"     | 精确到作曲家的和声/节奏/配器特征 |
| 可操作性 | 你需要自己从头实现     | 一键生成完整项目文件             |
| 学习价值 | 一次性回答             | 可持续扩展的知识库               |

## 💡 使用示例

### 示例1：快速创作游戏Boss战音乐

```text
用户: 我需要一个3分钟的Boss战背景音乐，要史诗感，像汉斯·季默那种

AI:
[✅ 触发music-composition skill]
[📖 加载composer-styles.md → 提取Zimmer风格模板]
[🎼 设计结构: 极简引子→层层叠加→音墙高潮]

正在创作《Boss战史诗》...

✅ 已生成:
📁 Boss战史诗.mid (2:58, 12轨道, fcp-midi格式)
📄 LABS配置指南.md (完整的音色分配+效果器参数)
📋 创作说明文档 (段落分析+情绪曲线)

主要特点:
- 调性: d小调 → F大调 (光明战胜黑暗)
- 编制: 弦乐ostinato + 铜管fanfare + 定音鼓滚奏 + 合成器pad
- 动态曲线: ppp(0:00) → ff(1:45高潮) → ppp(2:58淡出)
- Zimmer特色: 开放和弦、持续音积累、节奏驱动

```

### 示例2：模仿贝多芬写交响乐主题

```text
用户: 用贝多芬的风格写一个命运式的动机，然后发展成8小节的乐段

AI:
[✅ 触发skill]
[📖 加载composer-styles.md → Beethoven部分]
[📖 加载composition-theory.md → 动机发展技巧]

🎼 贝多芬式动机设计:

原始动机 (短-短-长):
└─ G3(八分) - G3(八分) - Eb3(四分) ← 命运般的冲击

发展变奏:
1. 移位: C4-C4-Ab3 (上行大三度，更明亮)
2. 扩大: G3(四分)-G3(四分)-Eb3(二分) (更沉重)
3. 倒影: G3-Bb3-D4 (变成上行，希望感)
4. 节奏变化: G3(休止)-G3(十六分)-Eb3(附点) (更紧凑)

已输出MIDI文件包含4个变奏版本。

```

### 示例3：为LABS制作完整配器方案

```text
用户: 我想在LABS里做一段悲伤的大提琴独奏，需要完整的MIDI和参数设置

AI:
[✅ 触发skill]
[📖 加载orchestration-rules.md → 大提琴特性]
[📖 加载composer-styles.md → 悲伤风格模板]

🎻 大提琴独奏《回忆》创作方案:

乐器选择:
├─ 主奏: LABS Strings - Cellos (Legato articulation)
├─ 伴奏: LABS Soft Piano (极简和弦)
└─ 氛围: LABS Strings - Violins (con sordino, 极弱震音)

MIDI细节:
├─ 大提琴: 音域G2-C4 (最佳情感区), 力度mp-mf with heavy rubato
├─ 表情CC11: 手动绘制大波浪形 (模拟呼吸感)
├─ 速度: 58-64 BPM (弹性速度)
└─ 和声: i - VI - III - VII - i (和声小调)

效果器预设:
├─ Reverb: Hall Large 3.5s, Mix 25%
├─ EQ: 低频-2dB@100Hz, 中频+1dB@1kHz, 高频+2kHz@8kHz shelf
├─ Compression: Ratio 3:1, Attack 20ms, Release 250ms
└─ Delay: 1/8 note, 15% mix (极淡的回声)

✅ 已生成完整MIDI + 参数文档

```

## 🔧 技术依赖

### 必需工具（已预装）

| 工具        | 版本要求         | 用途         |
| ----------- | ---------------- | ------------ |
| Python      | 3.11+ (推荐3.14) | 运行环境     |
| music21     | 9.x              | 音乐理论分析 |
| pretty-midi | 0.2.x            | MIDI处理     |
| midiutil    | 1.2.x            | MIDI生成     |

### 可选增强工具

| 工具               | 说明                       | 状态                              |
| ------------------ | -------------------------- | --------------------------------- |
| **fcp-midi**       | 更强大的语义化MIDI语言     | ⚠️ 需Python 3.13 (你的系统是3.14) |
| **mcp-score**      | MuseScore集成+MusicXML导出 | ⚠️ 同上                           |
| **Spitfire LABS**  | 免费电影配乐音色库         | ✅ 需单独安装                     |
| **REAPER/Ableton** | DAW用于渲染和混音          | ✅ 推荐任选其一                   |

### 关于fcp-midi兼容性的说明

**问题**: fcp-midi 及其依赖 fcp-core 目前最高支持 **Python 3.13** **你的环境**: **Python 3.14.4**

**解决方案**:

1. ✅ **当前方案**: 本Skill集成的 `symphony_generator.py` 覆盖80%常用功能
2. ⚠️ **进阶方案**: 安装Python 3.13虚拟环境以使用fcp-midi全部功能
3. 🔧 **贡献方案**: 向fcp-midi提交PR添加Python 3.14支持

## 🎯 能力边界

### ✅ 我能做得很好的事

- ✅ 从零创作完整的多轨道MIDI作品
- ✅ 模仿特定作曲家的风格特征
- ✅ 专业的管弦乐配器和平衡
- ✅ 输出可直接使用的标准文件格式
- ✅ 提供详细的技术参数和建议
- ✅ 教授音乐理论和作曲技巧

### ❌ 我做不到的事

- ❌ 直接播放音频（只能生成MIDI，需DAW渲染）
- ❌ 保证"好听"（美学主观性强）
- ❌ 实时交互调整（不能即时试听修改）
- ❌ 复杂爵士即兴或高度自由的音乐
- ❌ 合成人声（可写旋律但无法合成声音）

### ⚠️ 需要人工优化的地方

- ⚠️ 表情曲线细节（CC11绘制需在DAW中手工完成）
- ⚠️ 真实演奏感（MIDI量化后的机械感需人性化处理）
- ⚠️ 最终混音平衡（需根据实际听感调整各轨音量/声像）
- ⚠️ 效果器微调（混响/EQ/压缩参数需试听后优化）

## 📚 持续改进

### 如何扩展这个Skill？

向 `references/` 目录添加新内容即可：

| 新增内容类型 | 目标文件                 | 说明                         |
| ------------ | ------------------------ | ---------------------------- |
| 新作曲家风格 | `composer-styles.md`     | 分析新的作曲家特征并添加模板 |
| 新配器技术   | `orchestration-rules.md` | 添加新的乐器组合或特殊效果   |
| 新理论内容   | `composition-theory.md`  | 补充新的和声/曲式/技法知识   |
| 成功案例     | 新建 `examples/` 目录    | 存储优秀作品的代码供参考学习 |
| 用户反馈     | 更新SKILL.md             | 根据实际使用体验优化流程     |

### 贡献指南

欢迎提交Issue和Pull Request：

1. **Bug报告** → 描述复现步骤和期望行为
2. **新功能请求** → 说明使用场景和预期效果
3. **内容补充** → 新的作曲家/配器/理论知识
4. **风格模板** → 你喜欢的特定作曲家或流派
