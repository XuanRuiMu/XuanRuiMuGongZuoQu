# PDF截图自检指南

当需要PDF转换流水线、LibreOffice/PyMuPDF配置或截图分析细节时使用本文件。

## 为什么截图验证不可妥协

代码自检（`self_check.py`）操作docx XML树。它可以读取文本内容、运行属性和结构元素。但它**无法**看到：

| 问题                | 代码检查                        | 截图检查                |
| ------------------- | ------------------------------- | ----------------------- |
| Word批注/注释残留   | ❌ XML文本中不可见              | ✅ 以批注框形式可见     |
| 字体渲染异常        | ❌ 只能看到字体名称，看不到字形 | ✅ 显示实际渲染输出     |
| 图片缺失或错位      | ❌ 只能看到图片引用存在         | ✅ 显示图片是否实际出现 |
| 页面布局/页边距问题 | ❌ 无页面级视图                 | ✅ 显示实际页面布局     |
| 表格边框完整性      | ❌ 只能看到边框属性已设置       | ✅ 显示边框是否实际渲染 |
| 下划线视觉效果      | ❌ 只能看到underline=True标志   | ✅ 显示下划线实际外观   |

## 流水线架构

```text
DOCX输入
    │
    ▼
[LibreOffice --headless]
使用真实渲染引擎将DOCX转换为PDF。
这不是虚假转换——它使用与LibreOffice Writer中
渲染文档相同的引擎。
    │
    ▼
PDF输出（例如，522 KB，9页）
    │
    ▼
[PyMuPDF / fitz]
打开PDF并将每页渲染为指定DPI的光栅图像。
无需poppler依赖——PyMuPDF自带渲染器。
    │
    ▼
PNG截图（例如，200 DPI下每页1654×2339像素）

```

## 前置条件（一次性设置）

```bash
# LibreOffice — PDF转换引擎
winget install TheDocumentFoundation.LibreOffice

# PyMuPDF — PDF转图片（无需poppler）
pip install PyMuPDF

```

## 配置

### DPI选择

| DPI | 分辨率（A4） | 使用场景         |
| --- | ------------ | ---------------- |
| 96  | ~800×1130    | 快速预览，小文件 |
| 150 | ~1246×1757   | 标准质量         |
| 200 | ~1654×2337   | **论文推荐**     |
| 300 | ~2480×3508   | 印刷质量，大文件 |

论文验证推荐**200 DPI**：清晰到可以看到字体细节，文件大小可控。

## 仅通过截图才能发现的常见问题

1. **模版批注框**：Word将批注存储在单独的XML中。它们不会出现在`p.text`中，但在PDF中会渲染。注意页边距中的彩色框或方括号文本。

2. **图片链接错误**：docx可能通过关系ID引用图片，但如果实际图片数据损坏或缺失，代码能看到引用但截图显示破损图片图标。

3. **字体替换**：如果文档指定了系统未安装的字体，LibreOffice会替换为其他字体。这可能改变换行、字符宽度和整体布局。

4. **表格边框折叠**：边框属性可能在XML中已设置，但渲染引擎在特定条件下可能抑制它们（例如，相邻表格合并时）。
