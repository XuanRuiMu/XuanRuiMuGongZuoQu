# 模版操作细则

当需要run级操作细节、占位符模式或Unicode修复方法时使用本文件。

## Run级操作

### 黄金法则

python-docx将每个格式化文本段表示为段落内的一个`run`。一个run携带：文本、粗体、斜体、下划线、字体名称、字号、颜色等。

当你调用`paragraph.runs.clear()`后跟`paragraph.add_run(new_text)`时，新run获得**默认格式**——无粗体、无下划线、无自定义字体。所有模版格式都会丢失。

因此：

```python
for run in paragraph.runs:
    if needs_replacement(run.text):
        run.text = run.text.replace(old, new)

```

### 查找哪个run持有目标文本

模版字段经常跨多个run拆分。例如：封面标题可能是：

```text
run0: "题目：××" (粗体, 下划线)
run1: "×××××" (粗体, 下划线)
run2: "我的世界RPG玩法系统的设计与实现" (正常)

```

你必须检查每个run的文本来定位占位符：

```python
for i, p in enumerate(doc.paragraphs):
    for j, run in enumerate(p.runs):
        t = run.text
        if any(ph in t for ph in ['×', 'XXXX', 'xxxx']):
            print(f'P{i} run{j}: "{t}"')

```

### 下划线run与全角空格

模版使用带全角空格（`\u3000`）的下划线run作为签名行。在某些字体中，`\u3000`渲染为方框字符（`□`）。修复方法：

```python
for p in doc.paragraphs:
    for run in p.runs:
        if run.underline and '\u3000' in run.text:
            run.text = run.text.replace('\u3000', ' ')

```

## 占位符模式

最终输出中不得出现的占位符字符完整列表：

| 字符   | Unicode | 名称        | 常见上下文           |
| ------ | ------- | ----------- | -------------------- |
| ×      | U+00D7  | 乘号        | 封面标题中的"××××××" |
| □      | U+25A1  | 白色方块    | 全角空格渲染伪影     |
| \u3000 | U+3000  | 表意空格    | 签名行、间距         |
| xxxx   | ASCII   | 小写x占位符 | 通用占位符           |
| XXXX   | ASCII   | 大写X占位符 | 通用占位符           |

检测代码：

```python
PLACEHOLDERS = [
    ('\u00d7', '× 乘号'),
    ('xxxx', '小写x'),
    ('XXXX', '大写X'),
    ('\u25a1', '白色方块'),
    ('\u3000\u3000\u3000\u3000', '四个连续全角空格'),
]

for char, label in PLACEHOLDERS:
    for i, p in enumerate(doc.paragraphs):
        if char in p.text:
            # 例外：×在数学公式中是合法的
            if char == '\u00d7' and any(kw in p.text for kw in ['乘法', '系数', '式（']):
                continue
            print(f'发现{label}于P{i}: {p.text[:50]}')

```

## 批注/注释检测

Word批注与段落文本分开存储。它们不会出现在`p.text`中。通过XML检测：

```python
from docx.oxml.ns import qn
comments = doc.part.element.findall('.//' + qn('w:comment'))
print(f'发现{len(comments)}条批注')

# 删除所有批注：
for comment in comments:
    comment.getparent().remove(comment)

```

批注在PDF输出中可见，即使代码无法在段落文本中找到它们。这就是为什么PDF截图验证至关重要。
