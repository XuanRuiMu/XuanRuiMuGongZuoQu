# M06 生成艺术

## 1. 何时调用本模块

本模块负责通过代码直接生成视觉素材或实时艺术体验，而不是实现通用 UI 或套用现成图片。

调用场景：

- 用户要求“生成艺术”“算法艺术”“代码艺术”“流场”“粒子系统”“p5.js / p5js”
- 用户需要网站中的程序化背景、Hero 艺术图、数据可视化艺术化表达
- 用户希望生成可交互的数字画布（鼠标/参数驱动）
- 用户需要为网页创建静态装饰图，但不想使用现成图片库
- 用户想用 AI 生成前端可直接使用的图片素材（海报、插图、封面）

不归本模块处理：

- 3D / WebGL / Three.js → 走 M05
- 通用 UI 实现、组件组合、布局系统 → 走 M03
- 设计 tokens、色彩系统、字体系统 → 走 M02
- CSS / GSAP 驱动的 UI 动效 → 走 M04

## 2. p5.js / Canvas 生成艺术工作流

### 2.1 工作流总览

生成艺术前端实现分为两步：

1. **算法哲学定义**：说明美学运动名称、哲学、算法方向与预期气质。
2. **可交互作品实现**：基于 p5.js 在浏览器中运行，支持参数探索、种子导航和导出。

### 2.2 哲学先行

实现代码前，必须先定义算法哲学。输出形式根据交付物规模选择：

- **复杂作品或多文件项目**：单独输出 `.md` 文档。
- **单文件快速原型或简单作品**：将哲学说明嵌入 HTML 注释或简短的“关于”面板中。

哲学内容包含：

- **运动名称**：1-2 个词，如“有机湍流”“量子谐波”“递归低语”“场动力学”“随机结晶”。
- **哲学阐述**：2-6 段（单文件原型可压缩为 1-2 段），说明计算过程、涌现行为、数学之美如何转化为视觉。
- **算法方向**：使用的核心机制（噪声、粒子、力场、递归、L-system 等）。
- **工艺标准**：强调最终作品必须像由顶尖专家耗费大量时间打磨，而非随机堆砌。

### 2.3 模板起点

所有 HTML 交互作品应使用一个简洁的 Viewer 作为起点。优先使用项目已有的 `templates/viewer.html`；若不存在，则直接在 HTML 文件内实现：

- 全屏或居中画布
- 侧边栏或悬浮面板承载参数控件
- 项目主题风格或 Anthropic 品牌风格
- 响应式基础布局

### 2.4 参数结构

```javascript
let params = {
  seed: 12345,
  // 自定义控制项，如 particleCount、noiseScale、speed、colorShift
};

function setup() {
  createCanvas(1200, 1200);
  randomSeed(params.seed);
  noiseSeed(params.seed);
}
```

### 2.5 控件设计

侧边栏至少包含四组控件：

- **种子**：显示当前种子、上一个 / 下一个、随机、跳转。
- **参数**：与艺术相关的可变参数滑块或输入框。
- **颜色**：可选的调色板切换或色相偏移。
- **操作**：重新生成、重置默认值、下载 PNG。

## 3. 算法艺术哲学与运动命名

### 3.1 核心哲学

- **过程优于产物**：美从算法执行中涌现，每次运行因种子不同而独特。
- **参数即表达**：思想通过数学关系、力、行为、规则传达。
- **受控混沌**：随机性被噪声函数、边界条件、物理规则约束。
- **涌现复杂性**：简单局部规则叠加出整体复杂形态。
- **专家工艺**：反复打磨参数范围、色彩、构图，使作品具有“经过无数小时调试”的质感。

### 3.2 运动命名示例

| 名称     | 哲学                         | 算法表达                                         |
| -------- | ---------------------------- | ------------------------------------------------ |
| 有机湍流 | 受自然法则约束的混沌         | 多层 Perlin 噪声驱动的流场，粒子轨迹累积为密度图 |
| 量子谐波 | 离散实体展现波动干涉         | 网格粒子携带相位，按正弦波演化                   |
| 递归低语 | 跨尺度的自相似，有限中的无限 | 递归细分的分支结构                               |
| 场动力学 | 通过物质运动使不可见的力可见 | 数学函数或噪声构建的向量场                       |
| 随机结晶 | 随机过程结晶为有序结构       | 随机圆填充或 Voronoi 镶嵌                        |

命名应抽象、有诗意，且能指导后续代码实现。

### 3.3 创作过程

用户请求 → 算法哲学 → 代码实现：

1. 诠释用户意图，提取概念种子（subtle conceptual reference）。
2. 创建算法哲学（运动名称 + 4-6 段阐述）。
3. 用代码实现，90% 算法生成 + 10% 必要参数。
4. 设计匹配哲学的参数与 UI 控件。

## 4. 常见生成技术

### 4.1 流场（Flow Fields）

- 使用 `noise(x * scale, y * scale)` 计算每一点的向量角度。
- 粒子根据角度更新速度，形成有机曲线。
- 多层噪声叠加可增加流动复杂度。
- 技巧：用透明度累积轨迹，形成密度与深度。

### 4.2 粒子系统（Particle Systems）

- 每个粒子具有位置、速度、加速度、生命周期。
- 力可来自噪声场、吸引力点、排斥力点或边界反弹。
- 连接邻近粒子可形成网格或网络结构。
- 控制粒子数量以平衡视觉效果与性能。

### 4.3 噪声（Noise）

- 优先使用 Perlin / Simplex 噪声，而非纯随机。
- `noiseSeed(seed)` 保证可复现。
- 噪声维度可扩展到 2D、3D、4D，用于时间演化。
- 结合 `map()` 将噪声值映射到位置、颜色、大小、角度。

### 4.4 递归（Recursion）

- 适用于分支、分形、树形结构。
- 递归深度与角度、长度衰减系数决定形态。
- 可引入随机扰动避免机械感。
- 与噪声结合可生成自然生长的有机分支。

### 4.5 L-systems

- 使用字符串重写规则生成植物、海岸线、装饰图案。
- 步骤：定义字母表、产生规则、迭代次数、解释器（ turtle graphics ）。
- 适合创建几何化、重复但有机的形态。

### 4.6 其他常用技法

- **Voronoi 镶嵌**：随机点集的空间划分，形成晶体或细胞结构。
- **圆填充 / 泊松盘采样**：均匀分布的圆或点，避免过度重叠。
- **agents / boids**：模拟群体行为，如鸟群、鱼群。
- **反应扩散（Reaction-Diffusion）**：模拟图灵斑纹，适合有机纹理。

## 5. AI 生成图片在前端中的使用

### 5.1 适用场景

AI 生图用于需要真实感、复杂场景或特定风格的静态视觉素材，而非实时算法画布：

- 游戏截图风格的宣传图
- 竖版电影海报、活动封面
- 产品图、角色概念图
- 需要具体人物、物体、场景的插画

### 5.2 调用方式

根据当前环境选择可用的图片生成方式：

- 若存在项目内图片生成脚本或 MCP 工具，优先使用。
- 若环境支持 `视觉图片生成` skill 或外部 API（如 GPT-Image、Stable Diffusion），按该工具规范调用。
- 若无法生成图片，使用占位图、CSS 生成图形或请用户提供素材。

调用时建议参数：

- 质量：`auto`（`high` 容易超时）
- 格式：`b64_json` 或 URL
- 超时：180 秒

### 5.3 提示词规则

1. 主体描述用英文，模型理解更稳定。
2. 中文仅用于标题文字，放在提示词末尾，如 `Chinese title 燃烧之陨备战12.0`。
3. 提示词总长度控制在 180 字符以内，硬性上限约 220 字符（超过易超时）。
4. 中文每个字约占 3 字符（UTF-8）。
5. 避免冗余形容词：`dramatic`、`epic`、`highly detailed` 等。
6. 数量词用数字：`10 players` 优于 `many players`。
7. 避免同时出现 `game UI` 与 `Chinese text`，极易超时。

### 5.4 输出与展示

生成完成后必须：

1. 读取并展示生成的图片。
2. 报告文件名、大小、保存路径。
3. 使用 AskUserQuestion 询问用户是否满意及下一步计划。

静态 AI 图片若用于前端页面，需按 M02/M03 规则进行响应式处理、压缩与 lazy loading。

## 6. 集成到网页

### 6.1 Canvas 尺寸

- 默认画布推荐 1200×1200 或 1920×1080，视使用场景而定。
- 在网页中展示时，使用 CSS 控制显示尺寸，而非改变 canvas 内部像素尺寸。
- 响应式场景下监听 `windowResized()`，重新计算 `resizeCanvas()`。

```javascript
function windowResized() {
  resizeCanvas(container.clientWidth, container.clientHeight);
  // 必要时重新初始化
}
```

### 6.2 性能

- 限制粒子数量、递归深度、每帧重绘区域。
- 静态艺术可在绘制完成后停止 `draw()` 循环（`noLoop()`）。
- 使用 `pixelDensity(1)` 在高 DPI 屏幕上避免过度采样。
- 对高分辨率导出，临时放大画布并单独渲染，再恢复。
- 离屏时暂停动画循环以节省资源。

### 6.3 导出

- PNG：使用 p5.js 的 `saveCanvas()`。
- 视频 / GIF：如需录制，使用 p5.js 的 `saveFrames()` 或外部库，但不在本模块详述。
- 静态资源导出后按项目资源目录存放。

### 6.4 嵌入方式

- 作为全屏背景：canvas 绝对定位，z-index 低于内容。
- 作为 Hero 区域：限定容器尺寸，居中展示。
- 作为缩略图 / 卡片：预渲染一帧或导出静态 PNG 展示。
- 确保 canvas 不会捕获所有鼠标事件而影响页面交互，可设置 `pointer-events: none` 或仅在需要交互时开启。

## 7. 输出格式

本模块产物统一为以下三种形式之一或组合：

### 7.1 算法哲学文档

- 文件名：`{movement-name}-philosophy.md`
- 内容：运动名称、哲学阐述、算法方向、工艺标准、参数建议。

### 7.2 可交互生成艺术作品

- `index.html`：基于 viewer 模板的自包含页面。
- `sketch.js`：p5.js 生成算法与参数控件逻辑。
- 可选 `styles.css`：Viewer 样式补充。
- 支持：种子导航、参数调节、重置、下载 PNG。

### 7.3 静态视觉资源

- `.png` / `.pdf`：由算法画布导出或 AI 生成的静态文件。
- 用于直接嵌入网页、PPT、打印物料。
