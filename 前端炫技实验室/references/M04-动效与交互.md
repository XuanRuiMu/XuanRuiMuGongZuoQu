# M04 动效与交互

## 1. 何时激活本模块

当用户需求涉及以下任一方向时，在整体方案中激活 M04：

- 页面加载动画、入场编排、stagger reveal
- 滚动触发效果（视差、进度指示、元素显隐）
- 微交互：hover / focus / active / pressed / disabled 状态反馈
- 锚点定位驱动的跟随菜单、工具提示、下拉对齐
- 需要在 CSS 与 JS 动画库之间做技术选型
- 需要处理 `prefers-reduced-motion` 或减少动画对无障碍的影响

> 边界提示：
>
> - 设计 token、色彩、字体规则 → 见 M02
> - 布局、栅格、构图 → 见 M03
> - 3D、WebGL、Shader、粒子 → 见 M05
> - 3D 场景内部的相机移动、模型旋转、粒子效果、射线交互反馈由 M05 负责；M04 只处理 UI 层面动效（页面入场、卡片过渡、hover/focus/active 状态、Toast、弹层等）。
> - 深性能优化（Core Web Vitals、RAF 调度、渲染管线） → 见 M14
> - React/Next.js 组件动画细节 → 见 M07

## 1.1 增强现有页面时的边界

当本模块用于增强 M03/M05/M07 等模块已生成的页面时，必须遵守：

- **只添加动画层**：保留原有 HTML 结构、布局类（如 grid/flex 定位、col-span、row-start 等）和视觉样式。
- **不删除已有 class**：新增动画类时通过追加而非替换；如需条件类，使用 `classList.add` 或 `cn()` 合并。
- **动画 CSS 隔离**：动画相关样式放在独立注释块中，与原有样式清晰分隔。
- **回退优先**：对不支持 `animation-timeline` 的浏览器，使用 `IntersectionObserver` 或保持静态布局，不破坏页面可读性。

## 2. 2026 CSS 动画特性速查

以下特性在 2026 年已具备可用基线，优先在 CSS 层解决，避免过早引入 JS 动画库。

### 2.1 Scroll-driven Animations

使用滚动位置驱动动画时间线，无需监听 `scroll` 事件。

```css
@keyframes fade-in-up {
  from {
    opacity: 0;
    transform: translateY(40px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.section {
  animation: fade-in-up linear both;
  animation-timeline: view();
  animation-range: entry 10% cover 40%;
}
```

关键概念：

| 属性                                | 用途                             |
| ----------------------------------- | -------------------------------- |
| `animation-timeline: scroll()`      | 以最近可滚动祖先的滚动进度驱动   |
| `animation-timeline: view()`        | 以元素自身在视口中的可见进度驱动 |
| `view-timeline` / `scroll-timeline` | 命名时间线，供多个元素引用       |
| `animation-range`                   | 限制动画生效的滚动区间           |

原则：

- 滚动动画只用于装饰性增强，不传递关键信息。
- 避免在大量元素上同时启用，防止合成层爆炸。
- 移动端注意 `view()` 可能因地址栏伸缩产生抖动。

### 2.2 Anchor Positioning

让浮动元素相对于锚点自动定位，无需 JS 计算坐标。

```css
.anchor {
  anchor-name: --tooltip-anchor;
}
.tooltip {
  position: absolute;
  position-anchor: --tooltip-anchor;
  inset-area: top;
  justify-self: anchor-center;
}
```

适用场景：

- 工具提示、下拉菜单、弹出层、跟随指示器
- 需要随锚点滚动/缩放/移动而自动更新的场景

回退策略：

- 旧浏览器回退到固定相对定位或 JS 计算坐标。
- 使用 `@supports (anchor-name: --x)` 做特性检测。

### 2.3 `:has()` 与状态驱动动画

```css
.card:has(.card__media:hover) .card__title {
  transform: translateY(-4px);
}

.group:has(:focus-visible) {
  outline: 2px solid var(--color-focus);
}
```

- 用 `:has()` 在父级感知子元素状态，减少无意义 wrapper。
- `:focus-visible` 配合 `:has()` 实现键盘焦点环的聚合反馈。

### 2.4 Container Queries 与尺寸驱动动画

```css
.card {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card__title {
    font-size: clamp(1.25rem, 4cqi, 2rem);
  }
}
```

- 容器查询让组件动画基于自身空间而非视口宽度。
- 适合卡片、侧边栏、弹窗等可复用组件。

### 2.5 `@starting-style`

用于元素进入 DOM 或从 `display: none` 显示时的初始样式。

```css
.dialog {
  display: none;
  opacity: 0;
  transition:
    opacity 0.3s ease,
    display 0.3s ease;
  @starting-style {
    opacity: 0;
  }
}
.dialog[open] {
  display: block;
  opacity: 1;
}
```

- 解决 `display` 无法过渡的历史问题。
- 适用于 dialog、popover、toast、展开面板。

### 2.6 `sibling-index()`

获取元素在兄弟节点中的索引，用于自动生成 stagger delay。

```css
.list-item {
  animation: slide-in 0.4s ease-out both;
  animation-delay: calc(sibling-index() * 80ms);
}
```

- 替代 `:nth-child()` 的硬编码 stagger。
- 列表增删时自动重新计算，无需 JS。

### 2.7 CSS `if()`

条件函数，根据布尔值返回不同值。

```css
.button {
  padding: if(var(--is-compact), 0.5rem 1rem, 1rem 2rem);
}
```

- 减少为单个属性写多套规则的冗余。
- 适合响应式、主题变体中的微小差异。

### 2.8 Typed `attr()`

让 `attr()` 返回带类型的值，可直接用于计算。

```css
.progress-bar {
  width: calc(attr(data-progress type(<number>), 0) * 1%);
}
```

- 适合从 HTML 读取数值并参与动画/计算。
- 当前优先用于进度、计数器、数据可视化装饰。

## 3. CSS 动画 vs JS 动画库选型

| 场景                           | 推荐方案                     | 说明                                                     |
| ------------------------------ | ---------------------------- | -------------------------------------------------------- |
| 单个元素固定时长循环/入场      | 纯 CSS `@keyframes`          | 零依赖，渲染线程友好，适合 shimmer、glow、mask、呼吸效果 |
| 滚动触发、视差、进度           | CSS scroll-driven animations | 优先于 `scroll` + `requestAnimationFrame`                |
| 多元素复杂时间线编排           | GSAP                         | timeline、pin、scrub、stagger 强大，适合营销页、演示文稿 |
| React 组件级 enter/exit/layout | Motion (原 Framer Motion)    | AnimatePresence、layout、gestures 与组件生命周期契合     |
| 轻量 React 微交互              | Motion / CSS transitions     | 优先 CSS，复杂状态再用 Motion                            |
| 需要 seek/预览/确定性时间线    | GSAP 或 CSS + Adapter        | 视频/演示场景需要精确控制当前帧                          |

决策顺序：

1. 能否用 CSS 完成？→ 用 CSS。
2. 是否需要组件生命周期感知？→ 用 Motion。
3. 是否需要复杂时间线、pin、scrub？→ 用 GSAP。
4. 是否需要精确 seek 到任意帧？→ 用 GSAP 或带 adapter 的 CSS。

注意事项：

- CSS 动画元素必须在运行时前放入 DOM。
- 需要与时间轴对齐的元素，保留 `data-start` 属性。
- 优先 `animation-fill-mode: both`。
- 避免 wall-clock JavaScript 驱动 CSS 动画。

## 4. 微交互规则

### 4.1 状态覆盖

| 状态             | 目标         | 典型实现                                   |
| ---------------- | ------------ | ------------------------------------------ |
| hover            | 暗示可交互   | 颜色/透明度/阴影过渡，scale ≤ 1.02         |
| focus            | 键盘导航可见 | 高对比度 outline 或 ring，不依赖颜色 alone |
| active / pressed | 确认已触发   | 短暂 scale 0.98 或背景加深                 |
| disabled         | 明确不可操作 | 降低透明度 + 禁用 cursor                   |
| loading          | 防止重复提交 | 替换文本为 spinner，保持按钮尺寸稳定       |

### 4.2 时序与缓动

- 交互反馈时长：100ms–200ms。
- 状态切换：200ms–300ms。
- 复杂入场：400ms–800ms。
- 缓动：
  - 入场：`ease-out`（快速出现，缓慢落定）
  - 退场：`ease-in`（缓慢消失，快速离开）
  - 交互反馈：`cubic-bezier(0.4, 0, 0.2, 1)`

### 4.3 不要做的事

| 反模式                           | 问题                     | 修正                              |
| -------------------------------- | ------------------------ | --------------------------------- |
| hover 时改变 width/height/margin | 导致布局抖动             | 改用 transform: scale / translate |
| hover 时无反馈                   | 用户不确定元素是否可交互 | 添加颜色/阴影/透明度变化          |
| 过渡超过 500ms                   | 感觉迟缓                 | 控制在 300ms 内                   |
| 所有元素同时动画                 | 视觉噪音                 | 按重要性分级，保留焦点            |
| 依赖 scale 放大图片/卡片         | 可能遮挡相邻元素         | 预留安全空间或使用阴影暗示        |

## 5. 滚动触发与页面加载编排

### 5.1 滚动触发

- 优先使用 CSS scroll-driven animations。
- 需要 JS 时，使用 Intersection Observer 触发一次性类名切换，而非监听 `scroll`。
- 滚动动画只增强视觉节奏，不隐藏关键内容。
- 长页面可添加阅读进度条：

```css
.progress {
  animation: grow linear;
  animation-timeline: scroll();
  transform-origin: left;
}
@keyframes grow {
  from {
    transform: scaleX(0);
  }
  to {
    transform: scaleX(1);
  }
}
```

### 5.2 页面加载编排

- 定义一个清晰的「首屏注意力焦点」，避免所有元素同时争抢视线。
- 使用 stagger 让元素依次出现：

```css
.hero__title {
  animation-delay: 0ms;
}
.hero__subtitle {
  animation-delay: 120ms;
}
.hero__cta {
  animation-delay: 240ms;
}
```

- 或使用 `sibling-index()` 自动生成延迟。
- 关键内容在无动画情况下也必须立即可读（动画降级）。

### 5.3 动效密度

| 页面类型    | 建议                                 |
| ----------- | ------------------------------------ |
| 营销落地页  | 1–2 个高冲击力时刻 + 若干微交互      |
| SaaS 仪表盘 | 克制，以数据稳定呈现为主             |
| 演示文稿    | 可更戏剧化，但每张幻灯片保持统一节奏 |
| 表单/流程   | 仅保留必要反馈，减少分心             |

## 6. 可访问性：prefers-reduced-motion

所有动画必须考虑运动敏感用户。

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

更精细的处理：

```css
.hero__reveal {
  animation: fade-in-up 0.6s ease-out both;
}
@media (prefers-reduced-motion: reduce) {
  .hero__reveal {
    animation: none;
    opacity: 1;
    transform: none;
  }
}
```

原则：

- 必须保留媒体查询全局回退。
- 关键信息不依赖动画呈现。
- 自动播放的循环动画在离屏时暂停。
- 避免大幅度视差、快速闪烁、覆盖大面积屏幕的运动。

## 7. 常见反模式

动画性能规则（只动画合成属性、`will-change` 使用、layout thrashing 避免）见
[M14 性能优化 - 动画性能](M14-性能优化.md#3-动画性能)。

### 7.1 动画依赖用户事件切换 class

- 避免 hover 触发复杂的 keyframe 动画。
- 交互反馈用 `transition` 而非 `animation`。
- 复杂的 enter/exit 用 `@starting-style` 或 JS 动画库管理。

### 7.2 无意义的装饰动画

- 每个动画都应有目的：引导注意力、确认操作、解释状态变化。
- 避免「因为能动画所以动画」。
