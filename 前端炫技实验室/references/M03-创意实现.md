# M03 创意实现

> 定位：把设计系统转化为真实界面的实现指南。覆盖：栈选择、空间构图、背景细节、生产级代码、复杂组件架构、2026 新特性。边界：不包含设计 token 生成（M02）、动画实现细节（M04）、3D/WebGL（M05）、生成艺术（M06）、框架深度规则（M07）、AI
> SDK（M08）、验证流程（M11/M12）。

---

## 1. 何时调用本模块

在以下场景进入 M03：

- 用户要求“做一个网页 / 落地页 / 仪表盘 / 应用 / 组件”并需要产出可运行代码。
- 已完成 M02 设计系统，需要将颜色、字体、风格关键词落地为 HTML/CSS/JS。
- 需要决定技术栈、布局结构、背景处理方式、代码组织方式。

若任务仅停留在“给设计建议 / 生成配色 / 挑选字体”，应返回 M02；若涉及复杂动画、3D、生成艺术、React 深度规则，则分别转交 M04/M05/M06/M07。

## 2. 默认栈指南

未指定技术栈时，默认优先 `html-tailwind`。完整栈选择指南见 [M07 组件与框架](M07-组件与框架.md)。

按复杂度递增的简要决策：

| 场景 | 推荐栈 | 下一步 |
| --- | --- | --- |
| 单页静态 / 个人作品集 / 营销落地页 | `html-tailwind` | 继续在 M03 实现 |
| 多表单 / 局部交互 / 可复用组件 | `react` / `vue` / `svelte` | 按用户要求进入对应框架 |
| 多路由 / Dashboard / 复杂状态 / shadcn 组件 | `nextjs` + `shadcn/ui` | 进入 M07 进行多组件架构 |
| 3D / 生成艺术 / 复杂动画 | — | 分别进入 M05/M06/M04 |

### 栈选择原则

1. **最小够用**：能用 HTML/Tailwind 完成的，不用 React；能用 React 完成的，不用 Next.js。
2. **单文件偏好**：简单作品优先输出单文件 `index.html`，降低用户运行成本。
3. **CSS 变量统一主题**：无论哪一栈，都通过 CSS 变量承载 M02 给出的颜色与字体 token，禁止在代码里写死 Tailwind 裸值如
   `bg-indigo-500`。
4. **框架无关的审美规则不变**：不对称、留白、纹理、双模式等规则在所有栈中均需执行。

## 3. 布局与空间构图规则

### 3.1 核心原则

- 实现侧空间检查：每个页面至少应用 3 项差异化空间处理（不对称、重叠、破网格、对角线流动、杂志式分栏、有意留白）。
- 禁用模式与差异化技术完整列表见 [M02 设计系统 - 反 AI 同质化规则](M02-设计系统.md#6-反-ai-同质化规则)。

### 3.2 不对称与破网格

| 技术       | 做法                                         | 示例                                               |
| ---------- | -------------------------------------------- | -------------------------------------------------- |
| 偏移网格   | 主标题偏左 / 偏右，配图跨出容器边缘          | `ml-[-10vw]` 或 `translate-x-12`                   |
| 交错列     | 两列/三列不在同一水平线，奇偶项垂直错位      | `odd:mt-16 even:mt-0`                              |
| 破格出血   | 图片或装饰块突破最大宽度容器，延伸至视口边缘 | 使用 `w-screen` 或负 margin                        |
| 对角线流动 | 用倾斜分割线、斜向排列元素引导视线           | `clip-path: polygon(...)`                          |
| 杂志式分栏 | 大标题跨多列，正文窄栏，侧边注释             | CSS 多列或 Grid                                    |
| 重叠层次   | 文字压在图片上，卡片压在线条上               | 使用 `grid` 将多个元素放在同一单元格，配合 z-index |

### 3.3 留白与密度控制

- **宏观留白**：section 之间使用充足垂直间距（推荐 `py-24` 到 `py-40`），避免内容堆叠。
- **微观留白**：卡片内部、按钮内边距、图标与文字之间保持一致节奏。
- **密度对比**：高密度信息区（仪表盘、表格）与低密度呼吸区（hero、引言）交替出现。
- **边缘安全**：浮动元素、破格出血元素与视口边缘保留最小间距，移动端避免贴边。

### 3.4 响应式网格基线

```css
/* 移动优先 */
.grid-responsive {
  display: grid;
  gap: 1.5rem;
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .grid-responsive {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .grid-responsive {
    grid-template-columns: repeat(12, 1fr);
  }
}
```

### 3.5 Hero 是论点

Hero section 不只是视觉入口，它是整个页面论点的浓缩。用户应在 3 秒内理解：

1. **这是什么**：产品/服务/内容的核心定义。
2. **为谁而做**：目标用户或场景。
3. **为什么继续看下去**：独特价值主张。

**执行规则**：

- Hero 标题应是可独立传播的句子，而非口号式形容词。
- 视觉元素必须支持标题论点，而不是装饰。
- 若删除 Hero 后页面论点不变，则 Hero 设计失败。

**示例**：

| 反例 | 正例 |
| --- | --- |
| 欢迎来到未来 | 用 AI 在 10 秒内把会议录音转成结构化纪要 |
| 更快、更智能、更美好 | 自动化客户支持，首次响应时间从 4 小时降到 30 秒 |

### 3.6 结构即信息

页面结构本身就在传递信息。相同的内容以不同结构呈现，会被用户解读为不同的产品定位。

| 结构选择 | 传递的信息 |
| -------- | ---------- |
| 单栏长滚动 + 大图 | 叙事驱动、品牌故事、情感优先 |
| 三栏特性网格 | 功能清单、SaaS 产品、理性对比 |
| 杂志式分栏 + 侧边注释 | 编辑品质、深度内容、专业权威 |
| 仪表盘式高密度网格 | 工具属性、数据驱动、效率优先 |

**执行规则**：

- 先确定页面要传递的核心信息，再选择结构。
- 不因为「常见」而使用三栏网格；只有功能清单型内容才适合。
- 每个 section 的切换都应有信息递进，而非单纯视觉变化。

## 4. 背景与视觉细节

背景不是“装饰”，而是氛围与深度的来源。根据设计系统选择 1-3 种处理方式。

### 4.1 渐变

- 颜色选择必须来自 [M02 设计系统](M02-设计系统.md)，禁用通用蓝紫渐变等 AI 同质化模式。
- 使用品牌色或暖/大地色调；渐变方向可多样化：`to-br`、`to-tr`、锥形渐变、径向渐变。
- 多层渐变叠加可营造深度：底层柔和弥散 + 上层尖锐过渡。

### 4.2 噪点与颗粒

- 使用 SVG 滤镜或 base64 噪点纹理叠加，透明度控制在 `0.03-0.08`。
- 统一作用于整个页面或 section，降低数字感，增加印刷/胶片质感。

示例：

```css
.noise-overlay {
  position: fixed;
  inset: 0;
  pointer-events: none;
  opacity: 0.04;
  background-image: url("data:image/svg+xml,...noise...");
  z-index: 9999;
}
```
### 4.3 几何图案与网格线

| 图案 | 用途 | 实现 |
| --- | --- | --- |
| 点阵 | 科技感、数据感 | `radial-gradient(circle, ...)` 背景 |
| 细网格线 | 编辑/建筑/设计工具感 | `linear-gradient` 重复 |
| 对角线 | 动感、前卫 | `repeating-linear-gradient(...)` |
| 有机 blob | 自然、美妆、生活方式 | 使用 CSS `border-radius` 复杂值或 SVG |

### 4.4 纹理与材质

- 纸张、帆布、金属、毛玻璃等材质通过 `backdrop-filter`、`box-shadow`、`border`、`opacity` 组合表达。
- 纹理应微妙，不能干扰正文可读性。

### 4.5 阴影与边框

- 阴影处理遵循 [M02 设计系统 - 禁用模式](M02-设计系统.md#61-禁用模式)，禁用发光阴影。
- 优先使用中性色 subtle 阴影：`shadow-sm`、`shadow-md`。
- 边框可作为结构元素使用：`border-t`、`border-l` 强调分栏；深色模式下边框颜色需重新映射。

### 4.6 自定义光标与装饰

- 可交互区域使用 `cursor-pointer`。
- 自定义光标仅在强烈风格化需求下使用，且必须提供 fallback。

## 5. 生产级代码要求

### 5.1 通用代码质量

1. **语义化 HTML**：正确使用 `<header>`、`<nav>`、`<main>`、`<section>`、`<article>`、`<footer>`、`<button>`、`<a>`。
2. **CSS 变量管理主题**：

```css
:root {
  --color-bg: #ffffff;
  --color-surface: #f8f8f7;
  --color-text: #1a1a1a;
  --color-muted: #6b6b6b;
  --color-accent: #c25e00;
  --font-display: 'Unbounded', sans-serif;
  --font-body: 'Switzer', sans-serif;
}

.dark {
  --color-bg: #0d0d0d;
  --color-surface: #1a1a1a;
  --color-text: #f2f2f2;
  --color-muted: #999999;
  --color-accent: #ff8c42;
}
```
1. **双模式默认实现**：浅色为默认，深色通过 `dark` 类切换；切换按钮使用太阳/月亮 SVG；持久化 `localStorage`；首次尊重 `prefers-color-scheme`。
2. **可访问性基线**：所有图片有 `alt`；表单输入有关联标签；交互元素有可见 focus；颜色不是唯一状态指示器；尊重 `prefers-reduced-motion`。
3. **图标规范**：统一使用 Lucide / Heroicons / Tabler SVG；禁用 emoji；固定 `viewBox="0 0 24 24"` 与统一尺寸。
4. **字体加载**：使用 Google Fonts 或自托管，提供 `font-display: swap`；备用字体栈需匹配字形类别。
5. **响应式断点**：在 `375px`、`768px`、`1024px`、`1440px` 验证布局；移动端禁止水平滚动。

### 5.2 反 AI 同质化检查

设计层面的禁用模式与差异化技术完整说明见 [M02 设计系统 - 反 AI 同质化规则](M02-设计系统.md#6-反-ai-同质化规则)。

实现侧检查项：
- 至少 3 项差异化空间处理来自第 3 节。
- 背景、阴影、纹理等视觉细节来自第 4 节与 M02 设计系统。
- 色板、字体、图标来源已在第 5.1 节中约定。

### 5.3 性能与可维护性

- 图片使用适当格式（WebP/AVIF）与尺寸。
- 避免过度嵌套；Tailwind 类按布局 → 间距 → 外观 → 交互顺序排列。
- 复杂选择器优先使用 CSS 变量与类，不滥用 `!important`。
- 生产交付前清理调试代码、console.log、未使用资源。

## 6. 何时使用复杂多组件架构

以下场景跳出单文件 html-tailwind，启用 `Web作品构建` 式多组件架构（React + Tailwind + shadcn/ui，详见 M07）：

1. **多页面或路由需求**：需要客户端路由、动态路由、嵌套路由。
2. **复杂状态管理**：跨组件状态、表单状态、异步数据缓存、购物车、仪表盘筛选。
3. **大量使用现成 UI 组件**：Dialog、Sheet、Dropdown、Tabs、DataTable、Chart 等需要 accessible 行为。
4. **需要构建工具链**：TypeScript、热更新、打包优化、环境变量管理。
5. **团队协作或长期维护**：组件化、测试、类型安全成为硬需求。

### 简单 vs 复杂决策树

```
用户请求
├─ 单页静态 / 个人作品集 / 营销落地页
│  └─ html-tailwind 单文件
├─ 多表单 / 局部交互 / 可复用组件
│  └─ React/Vue/Svelte 单页或少量组件
├─ 多路由 / Dashboard / 复杂状态 / shadcn 组件
│  └─ Next.js/React + shadcn/ui 多组件架构（M07）
└─ 3D / 生成艺术 / 复杂动画
   └─ 分别进入 M05/M06/M04
```

## 7. 2026 新增 CSS 特性

在支持的浏览器中优先使用以下特性，实现更干净、可维护的创意布局。

### 7.1 CSS Grid lanes / masonry

- `grid-template-rows: masonry` 提供原生瀑布流，无需 JavaScript 库。
- 结合 `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))` 实现响应式 masonry。
- 回退：不支持浏览器回退到标准 grid 或 column-count。

```css
.masonry-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  grid-template-rows: masonry;
  gap: 1.5rem;
}
```
### 7.2 `@scope`

- 将样式限制在特定 DOM 子树，避免全局类名污染。
- 适用于 landing page 中独立 section 的局部样式。

```css
@scope (.hero-section) {
  :scope {
    background: var(--color-surface);
  }
  h1 {
    font-size: clamp(2.5rem, 6vw, 5rem);
  }
}
```
### 7.3 Typed `attr()`

- `attr(data-offset px)` 允许从 HTML 属性读取数值并带单位使用。
- 适合根据数据属性动态设置偏移、大小、颜色等，减少内联 style。

```css
.stagger-item {
  transform: translateY(attr(data-offset px, 0px));
}
```
### 7.4 `sibling-index()`

- 原生获取元素在兄弟节点中的索引，无需计数器或 JS。
- 用于交错布局、自动编号、渐进透明度。

```css
.stagger-item {
  opacity: calc(1 - sibling-index() * 0.05);
  transition-delay: calc(sibling-index() * 50ms);
}
```
### 7.5 特性使用原则

- 始终提供优雅降级：将新特性放在 `@supports` 中，或保证回退样式可接受。
- 不因炫技牺牲浏览器兼容性；根据目标用户群体决定 polyfill 策略。
- 2026 特性不替代 M04 的动画方案，而是布局与结构层面的补充。

## 8. 调用入口

当本模块被激活时，按以下顺序执行：

1. 确认已完成 M02 设计系统（或先调用 M02）。
2. 根据需求选择默认栈（html-tailwind 优先）。
3. 依据第 3 节设计空间构图，应用至少 3 项差异化技术。
4. 依据第 4 节处理背景与视觉细节。
5. 依据第 5 节保证生产级代码质量。
6. 若命中第 6 节条件，转交 M07 进行多组件架构实现。
7. 输出可运行代码，并标记后续进入 M04/M05/M06/M11/M12 的边界点。
