# M11 视觉验证

## 用途

在页面完成前端实现后，通过 Playwright 截图 + 人工/模型视觉审查，验证页面是否按设计稿正确渲染、三视口响应式是否合格、是否存在阻塞性视觉 BUG。

本模块只处理「看见的问题」，不涉及功能逻辑正确性（见 M12）、可访问性修复（见 M13）、性能优化（见 M14）。

## 触发条件

- 用户说：截图验证、检查网页渲染、验证 UI、visual verification、看看页面长什么样、审查设计、design
  review、UI 审查、视觉检查、设计质量、看看截图有什么问题
- 前端页面/组件刚写完，需要快速验证视觉质量
- 需要收集控制台渲染错误并判断其对页面的影响

## 前置条件

| 条件                | 说明                                                                                                               |
| ------------------- | ------------------------------------------------------------------------------------------------------------------ |
| 本地服务器运行      | 页面已可通过 URL 访问，例如 `http://localhost:3000/`、`http://localhost:5173/`、`http://127.0.0.1:8000/login.html` |
| Playwright MCP 可用 | Trae CN IDE 中已启用 `mcp_Playwright`                                                                              |
| 识图模型            | 当前环境模型 `Kimi-K2.7-Code` 支持读图；若不可用，仅输出截图路径并由用户查看反馈                                 |

## 执行流程

### 第 1 步：确认目标 URL

询问用户或根据上下文确定要验证的页面 URL。若 URL 未提供，则停止并追问。

### 第 2 步：三视口截图

使用 `mcp_Playwright` 依次截取桌面端、平板端、移动端三个视口。推荐目录为项目根目录下的 `.visual-verification/`。

#### 前置准备

1. 清理或归档 `.visual-verification/` 中的旧截图（或在文件名中加入时间戳），避免历史文件干扰判断。
2. 若目录不存在，先创建 `.visual-verification/`。

#### 视口配置

| 设备         | 宽度 | 高度 |
| ------------ | ---- | ---- |
| 桌面端（最小） | 1280 | 720  |
| 平板端       | 768  | 1024 |
| 移动端       | 375  | 812  |

桌面端 1280×720 是最小桌面视口，若项目面向宽屏用户，建议追加 1920×1080 视口截图。

#### 每个视口的操作

1. `playwright_navigate` 打开目标 URL，并传入对应 `width` 和 `height`
2. 等待页面主要元素渲染完成（必要时使用 `playwright_evaluate` 等待关键 DOM）
3. `playwright_screenshot` 保存 PNG，命名规则：
   - `{page}-desktop.png`
   - `{page}-tablet.png`
   - `{page}-mobile.png`
   - `{page}` 取自 URL 路径最后一段的 kebab-case，根路径默认 `home`；例如 `http://localhost:3000/dashboard` → `dashboard`，`http://localhost:3000/` → `home`。

#### 截图工具参数示例

`playwright_navigate`：

```json
{
  "url": "http://localhost:3000/",
  "browserType": "chromium",
  "width": 1280,
  "height": 720,
  "headless": false
}
```

`playwright_screenshot`：

```json
{
  "name": "home-desktop.png",
  "fullPage": true,
  "savePng": true,
  "downloadsDir": "d:\\XiTongWenJianJia\\ZhuoMian\\燃烧之陨我的世界服务端\\.visual-verification\\"
}
```

`playwright_resize`（切换视口时）：

```json
{
  "width": 768,
  "height": 1024
}
```

### 第 3 步：收集控制台错误

使用 `playwright_console_logs` 获取渲染相关错误：

```json
{
  "type": "error",
  "limit": 50
}
```

如果项目使用 React/Vue/Angular，再补充获取 `warning`
类型日志。将错误按「导致页面异常」「影响局部渲染」「可忽略」三类标记，只把前两类纳入问题清单。

### 第 4 步：读取截图并审查

使用 Read 工具读取三个 PNG 文件，对照下方审查清单逐项检查。

## 视觉审查清单

### 间距与构图

- [ ] 元素之间呼吸感足够，不过密也不过疏
- [ ] 整体布局有明确对齐关系（左/中/栅格）
- [ ] 移动端没有不必要的大片空白或过度拥挤
- [ ] 关键视觉重心位置合理

### 颜色

- [ ] 颜色数量克制，有明确主色与强调色
- [ ] 背景与文字对比度足够（仅做视觉判断，a11y 定量检测见 M13）
- [ ] 没有突兀、不协调或偏离设计系统的颜色
- [ ] 暗色/浅色模式切换后颜色保持一致性（如支持）

### 字体与排版

- [ ] 标题、正文、辅助文字层级清晰
- [ ] 字体选择符合整体调性
- [ ] 没有过小或过大的文字
- [ ] 中文显示正常，无乱码或字体回退异常
- [ ] 行高与段落间距舒适

### 内容与可见性

- [ ] 核心内容可见，无缺失
- [ ] 按钮、链接、输入框等交互元素可见
- [ ] 无明显的 Lorem ipsum 或占位内容残留
- [ ] 图片加载正常，无破图、拉伸或模糊

### 交互状态

- [ ] hover/focus/active 状态存在且合理
- [ ] 表单元素有明确的焦点指示
- [ ] 按钮有可点击的视觉暗示
- [ ] 下拉、弹层、遮罩等覆盖层位置正确

> 交互状态需要主动触发后再截图：对关键按钮/链接执行 `playwright_hover`，对表单输入执行 `playwright_click` 或 `playwright_evaluate` 设置焦点，然后截图。

### 响应式

- [ ] 桌面端、平板端、移动端布局均合理
- [ ] 移动端无横向滚动条
- [ ] 导航在小屏幕下可用（折叠/汉堡菜单等）
- [ ] 图片、卡片、表格等不会溢出容器

### 视觉 BUG

- [ ] 无空白页或大面积空白区域
- [ ] 无元素重叠、截断或超出边界
- [ ] 无文字被遮挡
- [ ] 控制台错误未导致页面明显异常

## 问题分级

| 等级 | 含义                                                            | 是否阻塞交付 |
| ---- | --------------------------------------------------------------- | ------------ |
| P1   | 严重视觉 BUG：空白页、核心内容不可见、严重错位/重叠、大面积截断 | 是           |
| P2   | 应修问题：布局偏差、内容缺失、交互状态缺失、响应式可用但体验差  | 建议修复     |
| P3   | 可优化：间距/颜色/字体微调、排版精修、边缘像素问题              | 可选         |

每个问题必须包含：

- 问题描述
- 出现位置（桌面/平板/移动/全局）
- 修复建议
- 优先级

## 修复与重测循环

1. 输出 P1/P2/P3 问题清单
2. 优先修复所有 P1，修复后立即重新截图验证对应视口
3. P2 问题修复后，选择代表性视口重新截图确认
4. P3 问题由用户决定是否在本次处理，处理后再验证
5. 直到所有阻塞性 P1 修复完毕，输出最终视觉验证报告

## 失败处理

| 问题                  | 处理方式                                                |
| --------------------- | ------------------------------------------------------- |
| mcp_Playwright 未启用 | 提示用户先在 Trae CN IDE 设置中启用 mcp_Playwright MCP  |
| 本地服务器未启动      | 提示用户提供启动命令或端口，确认后再继续                |
| 截图保存失败          | 检查 `.visual-verification/` 目录是否存在，不存在则创建 |
| 页面空白              | 标记 P1，继续完成剩余视口截图，最后统一输出问题清单     |
| 需要登录态            | 由用户提供登录步骤（cookie / 预登录脚本），完成后再截图 |
| 骨架屏/动态内容       | 先用 `playwright_evaluate` 等待目标 DOM 出现再截图      |

## 边界说明

- 本模块不验证功能逻辑，功能测试见 M12
- 本模块不做 WCAG 定量检测，a11y 修复见 M13
- 本模块不分析性能指标，性能优化见 M14
