# M10-幻灯片演示

## 1. 何时调用此模块

当用户需求涉及以下场景时，激活本模块：

- 创建 HTML 演示文稿、网页幻灯片、在线路演/演讲/汇报页面。
- 将 `.pptx` 文件转换为可在浏览器中运行的演示文稿。
- 增强或修改已有的 HTML 幻灯片（调整版式、增删内容、换肤）。
- 用户关键词：网页演示、HTML 幻灯片、在线演示、PPT 转网页、路演演示、演讲幻灯片、杂志风 PPT、瑞士风 PPT、Swiss
  Style、横向翻页。

如果用户只是需要通用网页设计、落地页或仪表盘，请转交 M03 创意实现模块；如果需要从零生成海报、单张视觉图片，请转交 M06 生成艺术模块。

## 2. 零依赖 HTML 幻灯片原则

每张幻灯片最终产物必须是一个可在浏览器中直接打开、无需构建工具的 HTML 文件：

- **单文件自包含**：所有 CSS、JS、字体 CDN 引用集中在一个 `.html` 文件中。
- **无 npm/构建步骤**：不依赖 webpack、vite、React 构建流程（风格 C/D 内部使用 CDN 字体与 Motion One 除外）。
- **字体走 CDN**：使用 Google Fonts 或 Fontshare 加载独特字体，禁止直接回退到系统默认字体。
- **图标使用 Lucide 或 SVG**：禁止在幻灯片中使用 emoji 作为图标。
- **离线兜底**：若使用 Motion One，优先通过 CDN 引入；如用户有离线播放需求，可单独提供本地 `motion.min.js` 副本，断网时至少保证内容可读。

## 3. 杂志风与瑞士风美学

本模块提供两种极致风格预设，一份 deck 只能二选一，不可混用。

### 3.1 风格 C · 电子杂志风（Editorial）

- **视觉锚点**：参考 _Monocle_ 杂志。
- **字体组合**：中文衬线标题（Noto Serif SC）+ 英文衬线标题（Playfair Display）。
- **色彩**：暖色调，5 套预设主题（墨水经典、靛蓝瓷、森林墨、牛皮纸、沙丘）。
- **背景氛围**：WebGL 流体/等高线/色散背景，hero 页透出，普通页几乎不可见。
- **布局**：10 种布局骨架，支持开场封面、章节幕封、数据大字报、左文右图、图片网格、两列流水线、悬念收束、大引用页、并列对比、图文混排。
- **硬规则**：
  - 衬线大标题必须使用模板中存在的 `h-hero` 类。
  - 图片使用 `height: Nvh` 控制，禁止使用 `aspect-ratio`（会撑破容器）。
  - 中文大标题不超过 5 个字，且加 `nowrap` 防止一字一行。

### 3.2 风格 D · 瑞士国际主义（Swiss）

- **视觉锚点**：参考 Massimo Vignelli + Helvetica Forever。
- **字体组合**：全程无衬线，典型为 Inter + Helvetica，大字使用 ExtraLight（字重 200）。
- **色彩**：单一 accent 色，4 套预设主题（IKB 蓝、柠檬黄、柠檬绿、安全橙）。
- **背景氛围**：WebGL 极细网格 + 点阵背景，信息驱动设计。
- **布局**：22 种版式 S01–S22，每个 `<section class="slide">` 必须写 `data-layout="Sxx"`。
- **硬规则**：
  - 全程无衬线、单一 accent 色，不允许多色高亮拼贴。
  - 无渐变、无阴影、无圆角（rule 横线除外），直角到底。
  - 字号对比 ≥ 8:1，卡片填充类型互斥（`card-ink` / `card-accent` / `card-fill` / `card-outlined` 不能混用）。
  - 7–8 页 deck 至少使用 6 个不同 S 编号；10 页以上至少 8 个。
- 不允许连续 3 页使用同种主体结构；主体结构按 `data-layout` 编号前两位分组（S01/S02 同属 hero 组，S07/S08 同属 grid 组，S12/S13 同属 media 组，以此类推）。开写 HTML 前先列 `页码 → data-layout → 结构组 → 选用理由 → 图片槽位` 草稿。

## 4. 视口适配规则

视口适配是幻灯片质量的第一道防线，每张幻灯片必须遵守：

- **容器定高**：每个 `.slide` 必须设置 `height: 100vh; height: 100dvh; overflow: hidden;`。
- **内容禁止滚动**：幻灯片内部永远不要出现滚动条。内容溢出时拆分幻灯片，而不是压缩。
- **流体尺寸**：所有字体大小与间距使用 `clamp(min, preferred, max)`，禁止使用固定 `px`/`rem`。
- **内容容器约束**：主体内容容器需要设置 `max-height` 限制。
- **图片约束**：图片最大高度使用 `max-height: min(50vh, 400px)`；图片幻灯片可放宽到最高 60vh。
- **高度断点**：针对 700px、600px、500px 三个高度断点做降级处理。
- **减少动效**：必须包含 `prefers-reduced-motion` 媒体查询支持。
- **避免反向 CSS 函数**：不要直接写 `-clamp()`、`-min()`、`-max()`（会被静默忽略），应使用 `calc(-1 * clamp(...))`。
- **内嵌基础样式**：将第 4 节视口适配规则直接内联到 `<style>` 中，不依赖外部 CSS 文件。

## 5. 幻灯片内容密度限制

超出限制时，必须拆分为多张幻灯片。

| 幻灯片类型 | 最大内容                                      |
| ---------- | --------------------------------------------- |
| 标题幻灯片 | 1 个主标题 + 1 个副标题 + 可选标语            |
| 内容幻灯片 | 1 个标题 + 4–6 个要点，或 1 个标题 + 2 段文字 |
| 特性网格   | 1 个标题 + 最多 6 张卡片（2×3 或 3×2）        |
| 代码幻灯片 | 1 个标题 + 8–10 行代码                        |
| 引用幻灯片 | 1 条引用（最多 3 行）+ 出处                   |
| 图片幻灯片 | 1 个标题 + 1 张图片（最高 60vh）              |

**修改现有幻灯片时**，需先计算现有元素数量并对照上表检查：

- 添加图片前确认该幻灯片仍有空间；否则将图片移到新幻灯片或减少其他内容。
- 添加文本时，每张幻灯片最多 4–6 个要点。
- 任何修改后验证：`.slide` 有 `overflow: hidden`；新元素使用 `clamp()`；图片有视口相对最大高度；内容在 1280×720 下适配。
- 如果修改会导致溢出，主动拆分内容并告知用户，不要等用户要求。

## 6. 幻灯片的动效与过渡规则

- **默认使用 HTML/CSS 动画**：简单入场、交错显示优先使用 CSS `animation-delay` 与 `@keyframes`。
- **仅在用户明确要求复杂编排入场时才启用 Motion One**：通过 CDN 引用，并在 HTML 元素上加 `data-anim` / `data-animate` 标记触发入场动效；同时保留本地兜底副本。不要为默认 deck 主动引入 Motion One。
- **聚焦高影响力时刻**：一个精心编排的页面加载配合交错显示，比分散的微交互更令人愉悦。
- **横向翻页交互**：键盘 ← → 翻页、空格、滚轮、触屏滑动；ESC 打开缩略图索引页；右下角展示导航点。
- **低功耗模式**：按 `B` 键切换 `body.low-power`，停止 WebGL/Canvas RAF 和 Motion 入场动画，右下角必须提示 `B 静态`。
- **减少动效兜底**：在 `prefers-reduced-motion: reduce` 下，关闭自动播放的入场动画，保留必要的状态切换。

## 7. 何时使用 WebGL 背景

WebGL 背景用于营造氛围和深度，但不应喧宾夺主。详细实现（shader、粒子、Three.js 初始化、性能优化）参考 M05
3D 与 WebGL 模块。

| 场景                | 是否推荐 | 说明                                                  |
| ------------------- | -------- | ----------------------------------------------------- |
| 风格 C 电子杂志风   | 推荐     | hero 页使用流体/等高线/色散背景；普通页可几乎不可见   |
| 风格 D 瑞士国际主义 | 推荐     | 使用极细网格 + 点阵背景，强化信息秩序感               |
| 数据汇报、工程汇报  | 可选     | 若强调冷静理性，可弱化或关闭 WebGL，仅用 CSS 几何图案 |
| 低端设备/省电场景   | 必须降级 | 提供 `B` 键低功耗模式，停止 RAF 渲染                  |
| 单页内容密度高      | 不推荐   | 复杂背景会干扰阅读                                    |

WebGL 背景仅作为氛围层，不应承载关键信息；关键内容必须能在背景完全关闭后仍然清晰可读。

### WebGL 开关与降级

- 通过 `<body data-webgl="on|off">` 控制是否启用 WebGL 背景，默认 `on`。
- 用户未明确要求氛围背景，或内容密度高、强调冷静理性时，设为 `data-webgl="off"`，改用 CSS 几何图案兜底：
  ```css
  body[data-webgl="off"] .bg-layer {
    background-image: radial-gradient(circle, var(--accent) 1px, transparent 1px);
    background-size: 24px 24px;
    opacity: 0.12;
  }
  ```
- 低功耗模式（按 `B` 键）自动设置 `data-webgl="off"` 并停止 RAF。

## 8. 导出与演示模式

生成后默认在浏览器中打开，随后提供三种可选动作：

### 8.1 在线演示模式

- 全屏浏览器播放，支持键盘、滚轮、触屏滑动导航。
- 风格 C/D 支持缩略图索引页（ESC 键）。
- 演讲者备注可写入 HTML 注释，不影响展示。

### 8.2 部署到在线 URL

- 使用 Vercel 进行免费托管，生成任何设备可访问的分享链接。
- 部署前确保本地图片/视频与 HTML 一起打包。
- 重新部署会更新同一 URL。

### 8.3 导出为 PDF

- 使用 Playwright 无头浏览器以 1920×1080 打开演示文稿，逐张截图并合并为 PDF。示例命令：
  ```bash
  npx playwright screenshot --viewport-size=1920,1080 --full-page deck.html slide-{n}.png
  # 或使用项目脚本：node scripts/export-deck-pdf.js deck.html
  ```
- 动画和交互不会保留，PDF 为静态快照。
- 幻灯片必须使用 `class="slide"`，导出脚本通过该选择器定位页面。
- 本地图片必须可通过本地 HTTP 服务加载（可用 `npx serve .` 或 Python `python -m http.server`）。
- 若 PDF 超过 10MB，询问用户是否需要压缩。

## 9. 工作流概览

1. **模式检测**：判断是新建演示、PPT 转换还是增强现有 HTML。
2. **内容发现**（新演示）：一次性 AskUserQuestion 询问目的、长度、内容准备度、是否需要内联编辑；如有图片则评估并共同设计大纲。
3. **风格发现**：根据氛围生成 3 个风格预览，或让用户直接选择风格 C/D 及主题色。
4. **生成演示文稿**：依据第 4 节视口规则、第 6 节动效规则与所选风格 C/D 规范，生成单文件 HTML。若项目未提供模板，使用本 skill 目录下 `templates/slide-deck-starter.html` 作为起点，填充 `<!-- SLIDES_HERE -->` 占位符，替换 `<title>` 中 `[必填]` 占位符。
5. **质量检查**：对照 checklist 逐项自检；风格 D 交付前运行校验脚本 `node scripts/validate-swiss-layout.js`（脚本检查 `data-layout` 数量、连续结构组、字号对比、card 类型互斥，见第 3.2 节）；必要时截图核对。
6. **交付与导出**：打开文件，告知位置、导航方式、自定义方法；询问是否需要部署或导出 PDF。
