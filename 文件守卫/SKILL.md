# 文件守卫 — 路径守卫 + Markdown守卫

当即将创建、修改或保存任何skill文件或.md文件时，**操作前**必须验证以下规则。任何一条不通过，禁止操作。

---

## 一、路径守卫

### 强制路径

所有skill文件必须保存在以下根目录下：

```text
D:\XiTongWenJianJia\ZhuoMian\燃烧之陨我的世界服务端\.agents\skills\
```

每个skill必须作为独立子目录存放，结构如下：

```text
.agents\skills\
├── skill-name\
│   ├── SKILL.md（必需）
│   └── 其他资源文件（可选）
├── another-skill\
│   ├── SKILL.md
│   └── ...
```

### 铁律 [P0]

#### 1. 路径验证 — 零偏移

创建或保存任何skill文件前必须确认：

- 目标路径以 `D:\XiTongWenJianJia\ZhuoMian\燃烧之陨我的世界服务端\.agents\skills\` 为根
- skill文件位于以skill名称命名的子目录中
- SKILL.md文件位于子目录根层级，不嵌套在更深的子目录中

**禁止**将skill文件保存到以下位置：

- 工作区根目录
- 任意临时目录
- 用户桌面或其他非标准路径
- `.agents\skills\` 以外的任何位置

#### 2. 操作拦截 — 创建/复制/下载全覆盖

以下操作必须触发路径检查：

| 操作类型      | 说明                                 |
| ------------- | ------------------------------------ |
| 创建新skill   | 从零编写SKILL.md时                   |
| 复制粘贴skill | 从外部来源复制skill内容时            |
| 下载skill     | 从网络或市场下载skill时              |
| 安装skill     | 通过find-skills或其他方式安装skill时 |
| 移动skill     | 将已有skill文件移动到新位置时        |

#### 3. 路径修正 — 自动纠正

如果检测到skill文件即将保存到错误路径：

1. **立即停止**当前写入操作
2. 计算正确路径：`D:\XiTongWenJianJia\ZhuoMian\燃烧之陨我的世界服务端\.agents\skills\{skill-name}\`
3. 确保目标子目录存在（不存在则创建）
4. 将文件保存到正确路径
5. 如果错误路径已有文件，迁移到正确路径后删除错误路径文件

### 例外

- skill内部的打包资源文件（scripts/、references/、assets/）可以位于skill子目录的更深层级
- 评估文件（evals/）可以位于skill子目录内

## 二、Markdown守卫

### 铁律 [P0]

#### 1. 编码完整性 — 零乱码

写入前必须确认：

- 文件内容以 **UTF-8 无BOM** 编码写入
- 内容中 **不存在** U+FFFD（替换字符 `�`）
- 内容中 **不存在** 截断的多字节序列（如中文字符UTF-8的3字节序列缺少第3字节）
- 如发现乱码，**禁止写入**，必须先修复或重建损坏内容

**自检方法**：写入后立即读回文件，验证无乱码字符。

#### 2. 标题格式 — ATX风格 + 空行包围

- 统一使用 `#` 风格标题（ATX），**禁止** setext风格（下划线风格）
- 标题 **前后** 必须各有一个空行
- 同一文件内 **禁止** 出现内容完全相同的标题

#### 3. 代码块 — 空行包围 + 语言标注

- 代码块开闭标记（` ``` `）**前后** 必须各有一个空行
- 代码块开标记 **必须** 标注语言（如 `bash`、`python`、`text`）
- 代码块内容 **禁止** 为空

#### 4. 列表 — 空行包围

- 列表 **前后** 必须各有一个空行
- 有序列表编号必须连续递增（1, 2, 3...），禁止跳跃

#### 5. 表格 — 紧凑风格 + 空行包围

- 表格使用 **compact** 风格：管道符 `|` 与内容之间 **无空格**
- 表格 **前后** 必须各有一个空行
- 分隔行使用 `|---|` 格式（短横线两侧无空格）
- 同一表格各列宽度无需对齐，但风格必须统一（全部compact）

## 操作前3秒规则

执行任何skill文件写入/创建/下载/复制操作，或.md文件写入操作前，必须回答：

### 路径检查

1. 目标路径是否以 `D:\XiTongWenJianJia\ZhuoMian\燃烧之陨我的世界服务端\.agents\skills\` 开头？ → 否则修正路径
2. skill是否有独立的子目录？ → 否则创建子目录
3. SKILL.md是否位于子目录根层级？ → 否则调整位置

### 格式检查

1. 内容中是否有乱码/替换字符？ → 有则禁止写入
2. 所有标题是否前后有空行？ → 无则补空行后再写入
3. 所有代码块是否前后有空行且标注语言？ → 无则补齐后再写入
4. 所有列表和表格是否前后有空行？ → 无则补空行后再写入
5. 表格是否使用compact风格（管道符与内容间无空格）？ → 否则改为compact风格后再写入
6. 同一文件内是否有内容完全相同的标题？ → 有则禁止写入，必须修改标题使其唯一
7. 有序列表编号是否从1开始连续递增？ → 否则重编号后再写入

**全部通过** → 允许操作 **任一不通过** → 修复后重新检查，通过后再操作
