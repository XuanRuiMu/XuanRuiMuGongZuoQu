# SKILL.md 编写指南

## 解剖结构

```
skill-name/
├── SKILL.md（必需）
│   ├── YAML frontmatter（name、description 必需）
│   └── Markdown 正文
└── 打包资源（可选）
    ├── scripts/ - 可执行代码
    ├── references/ - 按需加载的文档
    └── assets/ - 输出用文件
```

## 渐进披露

| 层级         | 内容               | 加载时机           |
| ------------ | ------------------ | ------------------ |
| 元数据       | name + description | 始终在上下文       |
| SKILL.md正文 | 指令               | 触发时加载，<500行 |
| 资源文件     | 脚本/参考          | 需要时加载，无限制 |

## description 写法

- 最多1024字符
- 第三人称
- 第一句：做什么
- 第二句：Use when [触发条件]
- 后续：触发场景 + 触发词
- 为避免欠触发，描述可稍微"主动"

## 正文风格

- 使用祈使句
- 解释原因，避免大写MUST
- 包含具体示例
- 大文件拆到 references/
- 引用文件时说明加载时机

## 示例模板

```markdown
---
name: skill-name
description: >
  [做什么]。Use when [触发器]。 触发场景：[场景列表]。 当用户说"[触发词]"时触发此技能。
---

# 技能名称

## 适用场景

- [场景1]
- [场景2]

## 工作流

### 阶段1

1. [步骤1]
2. [步骤2]

## 铁律

| 规则     | 内容       |
| -------- | ---------- |
| [规则名] | [规则内容] |
```
