# 联网研究方法

本文档详细说明阶段2"联网研究"的执行方法，借鉴 deep-research 和 research-web 的工作流。

## 研究子问题分解

将冷门任务拆解为3-7个研究子问题，确保覆盖任务的各个维度。

### 标准子问题模板

| 维度     | 子问题示例                               |
| -------- | ---------------------------------------- |
| 任务定义 | 这个任务是什么？属于什么领域？           |
| 标准做法 | 业界标准做法是什么？有哪些主流方法？     |
| 工具选择 | 有哪些工具/软件/库可以完成？各自的优劣？ |
| 操作步骤 | 具体操作步骤是什么？先后顺序？           |
| 注意事项 | 有哪些坑、风险、常见错误？               |
| 环境要求 | 需要什么环境、依赖、权限？               |
| 验证方法 | 如何判断任务成功完成？                   |

### 分解原则

- **MECE原则**：子问题之间相互独立、完全穷尽
- **由浅入深**：先理解概念，再找方法，最后看细节
- **任务导向**：每个子问题的答案都应直接服务于任务完成

## 多波探索搜索

借鉴 research-web 的多波探索模式，确保研究深度。

### 波次策略

| 波次   | 目标     | 搜索量            | 触发条件                  |
| ------ | -------- | ----------------- | ------------------------- |
| 波次1  | 广度覆盖 | 每子问题2-3个查询 | 始终执行                  |
| 波次2  | 深度补充 | 针对差距3-5个查询 | 波次1后有关键子问题未覆盖 |
| 波次3+ | 专项深挖 | 针对特定缺口      | 仍有关键信息缺失          |

### 波次1：广度搜索

对每个子问题执行2-3个WebSearch查询：

```text
查询1：clean C drive junk files Windows
查询2：Windows C盘清理工具 方法
查询3：free up C drive space Windows 11
```

**查询技巧**：

- 使用具体关键词，避免过于宽泛
- 尝试中英文双语查询（中文任务也搜英文，英文资料通常更全）
- 混合通用查询和新闻查询（获取最新信息）
- 加入版本号/年份（确保信息时效性）

### 收集发现

对每个搜索结果记录：

- 来源URL
- 关键信息摘要
- 数据点（具体数值、步骤、命令）
- 置信度初评

### 评估差距

波次1完成后评估：

- 哪些子问题已充分覆盖？（有2+独立来源）
- 哪些子问题信息不足？（只有1个来源或无来源）
- 是否发现了新的需要研究的子问题？

### 波次2+：深度补充

针对信息不足的子问题启动新波次：

- 调整关键词，使用更具体或替代术语
- 搜索特定工具/方法的官方文档
- 搜索常见问题和解决方案

## 深度阅读关键源

不依赖搜索摘要，对最有价值的URL用WebFetch获取完整内容。

### 选择深度阅读源的优先级

1. **官方文档**（最高优先级）— 工具/方法的官方说明
2. **权威教程** — 知名技术博客、教育平台
3. **GitHub README** — 工具的详细说明
4. **社区讨论** — Stack Overflow、Reddit、论坛
5. **个人博客** — 实战经验分享

### 深度阅读数量

- 短任务（1-2步）：3个关键源
- 中等任务（3-5步）：5个关键源
- 复杂任务（5+步）：7-10个关键源

## 多源交叉验证

关键信息必须多源验证，避免单一来源的错误信息。

### 验证规则

| 信息类型      | 最少来源数 | 验证要求             |
| ------------- | ---------- | -------------------- |
| 关键操作步骤  | 2          | 步骤一致或互补       |
| 具体命令/代码 | 2          | 命令一致             |
| 风险警告      | 2          | 风险被独立提及       |
| 性能数据      | 2          | 数据接近或有合理解释 |
| 个人观点      | 1          | 标注"单源未验证"     |

### 置信度标注

- **高**：3+独立来源一致
- **中**：2个独立来源一致
- **低**：仅1个来源，或来源间有矛盾

## 搜索工具使用

### WebSearch 使用要点

- `num` 参数：默认5，关键搜索可设8-10
- `query` 参数：使用完整自然语言问题或精确关键词
- 中英文双语搜索：同一子问题用中文和英文各搜一次

### WebFetch 使用要点

- 优先fetch官方文档和权威源
- 认证/私有URL会返回空，改用WebSearch
- 内容过长会被截断，关注关键部分

## 时效性要求

- 优先最近12个月的资料
- 工具/软件版本信息确认最新
- 标注资料的发布时间
- 过时信息标注"可能过时，需验证"
