# 任务规划模板

本文档提供阶段4"重新规划"的结构化模板和制定方法。

## 方案制定原则

1. **研究发现驱动**：方案必须基于阶段2的研究发现，不凭空设计
2. **工具优先**：优先使用阶段3发现的高质量工具，减少手动操作
3. **可验证**：每个步骤都有明确的验证方法
4. **风险标注**：标注每个步骤的风险等级和缓解措施
5. **决策点明确**：标注需要用户确认的关键决策点

```markdown
## 步骤设计要点

### 步骤粒度

- **单步操作**：一个命令或一组连续操作
- **可验证**：每步完成后能立即验证
- **可回滚**：如可能，每步都有回滚方法
- **有依赖**：明确标注步骤间的依赖关系

### 风险等级定义

| 等级 | 定义               | 示例                       | 处理方式         |
| ---- | ------------------ | -------------------------- | ---------------- |
| 低   | 可逆、无破坏性     | 读取文件、查询信息         | 直接执行         |
| 中   | 有限影响、可恢复   | 修改配置、安装软件         | 执行后验证       |
| 高   | 不可逆或大范围影响 | 删除文件、格式化、系统修改 | **必须用户确认** |

### 决策点识别

以下情况必须标注为决策点：

- 多种方案选择（如选工具A还是工具B）
- 不可逆操作（删除、覆盖、格式化）
- 影响系统的操作（修改注册表、系统服务）
- 需要用户提供信息（密码、路径、选择）
- 方案偏离原计划

## 方案确认流程

1. **生成方案文档**：按模板填写完整方案
2. **用AskUserQuestion展示摘要**：
   - 任务目标
   - 主要步骤数
   - 关键决策点
   - 风险等级
   - 预计产物
3. **询问用户**：
   - 是否同意执行？
   - 是否需要调整某部分？
   - 对决策点的预先决策？
4. **获确认后进入阶段5自动执行**

## 方案调整规则

- 用户要求调整时，只调整指定部分
- 调整后重新确认
- 执行中发现方案问题，暂停报告，获确认后调整
- 不自行扩大或缩小任务范围
```
