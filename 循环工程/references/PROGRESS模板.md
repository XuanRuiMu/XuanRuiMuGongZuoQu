# PROGRESS.md 模板

循环工程的精简状态文件。只保存当前推进所必需的信息；已完成项只留一行摘要；不写历史、过程、文件清单。

---

## 模板内容

```markdown

## 元信息

| 字段     | 值                                |
| -------- | --------------------------------- |
| 任务     | [任务名称]                        |
| 状态     | 进行中 / 已完成 / 已熔断 / 已阻塞 |
| 循环计数 | 0                                 |

## 目标

[一句话描述要达成的目标]

- [ ] 条件3：[如：构建成功]
```

## 压缩规则

1. 功能点一旦变为"已完成"，除一行状态摘要外，删除该功能点的详细验收标准、涉及文件、历史记录、备注
2. "已完成"小节每行不超过一行，格式：`{ID} 已完成：{一句话摘要}`
3. 不写入"循环历史摘要"、"修改文件清单"、"详细决策日志"
4. "当前决策"只保留对待处理功能点仍有影响的条目；已落实或已失效的决策立即删除
5. 每轮更新后立即删除因状态变化而失效的内容

## 填写要点

1. **停止条件必须可机器验证**：不能写"代码优化好了"，要写"所有测试通过且lint零报错"
2. **功能点粒度要粗**：一个功能点 = 一个完整可验证单元，不要拆得太细
3. **循环计数器要准确**：每轮循环+1，用于判断是否触发总循环熔断；若本轮并行派发多个无依赖/无文件冲突的子代理，仍只计 1 轮（Orchestrator 一次循环周期）
4. **只保留当前状态**：PROGRESS.md不是历史档案，不存过程
5. **H1 标题反映当前任务目标**：PROGRESS.md 的 H1 标题应随任务目标变化，回归测试不得对标题文字做精确匹配
