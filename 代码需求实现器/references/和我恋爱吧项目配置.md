## 开发验证注意事项

### Playwright MCP 浏览器验证规则

- **Playwright MCP 的浏览器界面刷新后就是最新界面，不存在老旧缓存问题**
- 修改代码后，必须使用 Playwright MCP 浏览器实际打开页面截图查看，确认更改是否生效
- **不能仅依赖 Vite HMR 或终端命令的布局数据来判断**——终端命令可能读取到的是旧服务器实例的缓存状态
- 需要有识图能力的 AI 自行通过截图/视觉检查来确认 UI 更改到底有没有生效
- 如果修改后页面未变化，**重启 Vite 开发服务器**（`npm run dev -- --force`）通常能解决 HMR 缓存问题
- 前端开发端口可能因占用而变化（5173→5174等），需注意当前实际端口

### Docker 生产环境部署规则

- **每次前端代码修改完成后，必须将更改持久化到 Docker 容器化生产环境**
- 部署命令（在项目根目录执行）：
  1. `docker-compose build frontend --no-cache` — 重建前端镜像（含 vue-tsc 类型检查 + vite 生产构建）
  2. `docker-compose up -d frontend` — 使用新镜像重启前端容器
- 前端容器名：`lovewithme-frontend`，映射端口 80(HTTP) / 443(HTTPS)
- Docker Compose 文件：项目根目录 `docker-compose.yml`
- 前端 Dockerfile：`frontend/Dockerfile`（多阶段构建：node:20-alpine 构建 → nginx:alpine 运行）
- **此规则为强制执行项，不可跳过**
